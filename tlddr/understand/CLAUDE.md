# tlddr/understand - Module Index

**Last Updated:** 2026-07-03

The deterministic Understand toolkit (stage 2, merged and proven end-to-end). The host agent does comprehension, section-tagging, and edge proposal and hands enrichment JSON to the CLI; these helpers do everything deterministic: build the slice the agent reads, score confidence, validate edges and section tags (machine-trust), derive triage, assemble a `Node`, render the vault with section-coverage + isolated-node surfaces. The node is an overlay over the `ExtractedDoc` store, never a content clone. No model calls here. The two host-agent procedures live in `skills/` (`generate-sections`, `understand`).

## Files Overview

### slice.py
**Purpose:** Reduce an ExtractedDoc to a bounded slice the agent reads.
**Key Functions:** `build_slice(doc, max_chars=8000) -> str` (title + structure markers/headings + warnings + capped content sample)
**Constants:** `_MARKER` (page/sheet/heading regex), `_MAX_STRUCTURE_LINES=60`, `_MAX_WARNINGS=20`

### confidence.py
**Purpose:** Script-derived extraction confidence (the LLM can't judge fidelity it didn't perform).
**Key Functions:** `extraction_confidence(doc) -> Confidence` (proportional: geospatial->HIGH, truncated xlsx->MEDIUM, no-pages docx->HIGH, else text-fraction with 0.9/0.5 thresholds)

### triage.py
**Purpose:** Deterministic traffic-light derivation (never hand-set).
**Key Functions:** `derive_triage(extraction, interpretation, questions) -> Triage` (blocking q->RED; min==LOW->RED; min==MEDIUM or any q->AMBER; else GREEN)
**Constants:** `_ORDER` (LOW<MEDIUM<HIGH)

### edges.py
**Purpose:** Machine-trust edge validation.
**Key Functions:** `validate_edges(proposed, known_node_ids, source_id) -> (valid, dropped)` (drops unknown-target and self-links; dedupes same target+relation)

### sections.py
**Purpose:** Load/validate the user-provided `sections.json` and validate section tags (machine-trust, mirrors edges). Node-free — it is about the section-spec artifact only.
**Key Functions:**
- `load_sections(path) -> list[Section]` - parse + validate (raises ValueError on duplicate ids or unknown parent refs)
- `section_ids(sections) -> set[str]`
- `validate_section_tags(tags, known_ids) -> (valid, dropped)` - keeps known unique in order, drops unknown
**Dependencies:** json, tlddr.models (Section)

### node_render.py
**Purpose:** Render a Node as markdown (YAML frontmatter overlay + readable body).
**Key Functions:** `render_node_markdown(node) -> str` (frontmatter via yaml.safe_dump incl. report_sections; body has description + `[[id]]` wikilinks + _triage.md pointer; never source content)
**Dependencies:** yaml (pyyaml)

### render.py
**Purpose:** Render the vault index and triage surfaces (the deterministic triage backbone).
**Key Functions:**
- `render_index(nodes) -> str` - markdown table (wikilink, doc_type, triage), sorted by id
- `section_coverage(nodes, sections) -> dict[str, list[str]]` - section id -> tagged node ids
- `isolated_nodes(nodes) -> list[str]` - nodes with no outgoing edge and not the target of any edge
- `_display_triage(node, questions) -> Triage` - re-derives a node's triage from its still-unresolved questions only, so a resolved blocking question visibly de-escalates the node on render without mutating the stored `node.triage`
- `render_triage(nodes, questions, sections=None) -> str` - nodes grouped Red/Amber/Green by `_display_triage`; `## Section coverage` (no-evidence called out, only when sections given) + `## Isolated documents`, both before the question sections; questions split into `## Open questions` (unresolved, with `> answer:` slots) and `## Resolved questions` (resolved, showing disposition + the recorded answer, no slot)
**Constants:** `_GROUP_ORDER`, `_GROUP_TITLE`

### commit.py
**Purpose:** Assemble a validated Node from the agent's enrichment dict + the source doc.
**Key Functions:** `build_node(enrichment, doc, known_node_ids, known_section_ids=frozenset()) -> (Node, dropped_edges, dropped_section_tags, questions)` (computes extraction confidence, validates edges + section tags, derives triage, populates report_sections; questions tagged raised_by="understand")
**Dependencies:** confidence, edges, sections, triage, tlddr.models
